# CLAUDE.md

`@posthog/brand` ÔÇö an npm library that bundles PostHog's brand assets (hedgehog
illustrations, logos, crests, and brand colors) as React components, raw SVGs, PNG
URLs, and color tokens. Fully offline; no runtime CDN.

## Data flow

```
Figma brand book  --sync.ts-->  assets/<namespace>/  --codegen.ts-->  src/generated/  --tsdown-->  dist/
```

- **`scripts/sync.ts`** ÔÇö the only networked script. Walks the configured Figma pages
  (`scripts/lib/figma-pages.ts`), renders every component to BOTH an optimized inline SVG
  (SVGO, `scripts/lib/svg.ts`) and an optimized PNG (pngquant palette-quantization + oxipng
  lossless recompress, `scripts/lib/png.ts`) via the Figma API, and writes `assets/`.
  Incremental: re-renders only assets whose Figma node subtree hash
  changed since the last sync. Needs `FIGMA_TOKEN` (a Dev/Full-seat PAT ÔÇö Viewer seats are
  ~6 req/month). Run with `pnpm sync` (`--force` re-renders all, `--check` reports drift).
  Colors are NOT synced ÔÇö they are a fixed, hand-maintained palette in `static/colors.ts`.
- **`scripts/lib/figma.ts`** ÔÇö rate-limit-aware Figma REST client (serialized, spaced by
  `FIGMA_MIN_INTERVAL_MS`, backoff on 429/5xx, batched `/images` renders). Only the API
  endpoints are metered; downloading rendered S3 assets is free.
- **`assets/`** ÔÇö committed source of truth for illustrations: `<namespace>/catalog.json`,
  `<namespace>/vectors/*.svg`, `<namespace>/png/*.png`, and `sync-state.json` (keyed
  `<namespace>/<slug>`).
- **`static/colors.ts`** ÔÇö the hardcoded brand palette (`BrandColors`), hand-maintained
  source of truth for colors. Edit a value here (and commit a changeset) to change the
  published palette; `codegen.ts` reads it.
- **`scripts/codegen.ts`** ÔÇö pure transform of `assets/` + `static/colors.ts` Ôćĺ
  `src/generated/`. Gitignored output; regenerated by `prebuild`/`pretest`. Run with
  `pnpm codegen`. Applies the rename overrides in `scripts/lib/renames.ts` (see
  Conventions) as it emits.
- **`scripts/lib/renames.ts`** ÔÇö manual slug/name overrides for assets whose Figma-derived
  names read poorly. A presentation layer applied at codegen time, offline.
- **`src/`** ÔÇö handwritten runtime + entry points; `src/generated/` is machine-written.
- **`src/logo/`** ÔÇö the `<Logo>` component (`./logo` export). NOT Figma-synced or codegen'd:
  the logo is a small, stable mark, so its SVG geometry is inlined in `src/logo/geometry.ts`
  and exposed as one generic, parametric compound component (`variant`/`layout`/`color`,
  plus `Logo.Logomark`/`Logo.Wordmark`). Hand-maintained ÔÇö edit it directly (with a changeset).

## Conventions

- Namespaces: `hoggies`, `crests` (Figma-synced illustrations) + `colors` (data) + the
  handwritten `logo` (see `src/logo/`). The synced namespaces are defined in `src/types.ts`
  (`NAMESPACES`) and mapped to Figma pages in `scripts/lib/figma-pages.ts`; `logo` is neither.
- Component name carries the family word + PascalCase(slug): hoggies prefix it
  (`Hedgehog<Name>`); crests append it (`<Name>Crest`) so the name reads the
  same in every crest export. SVG string export = lowerFirst(component)+`Svg`, PNG URL
  export = lowerFirst(component)+`Png` (e.g. `arrayCrestSvg`, `hedgehogDoctorHogPng`).
- **Crest tiers**: crests come in two sizes ÔÇö `full` and `mini` ÔÇö keyed off the Figma
  `Property 1` variant (`figma-pages.ts` `tierVariantProp`) and lifted into `AssetMeta.tier`
  rather than the slug. Each tier is its own **export group** (`ASSET_GROUPS` in `types.ts`):
  a subpath (`crests/full`, `crests/mini`), a generated dir (`src/generated/crests/<tier>/`),
  and on-disk sources under `assets/crests/<tier>/{vectors,png}/`. Both tiers share one
  `assets/crests/catalog.json`, so a crest slug is unique only within a (namespace, tier).
  Minis keep a trailing `Mini` in their identifiers (`ArrayCrestMini`, `arrayCrestMiniSvg`).
  The combined `@posthog/brand/crests` barrel (`src/generated/crests/index.ts`, one
  `src/generated/crests/<slug>.tsx` per slug) re-exposes them as a **compound component**
  `<Name>Crest` ÔÇö base is the full illustration, `.Mini` (when a mini exists) is the badge,
  attached via `Object.assign`. `writeCombinedCrests` in `codegen.ts` emits it after the
  group loop; the tier-specific `crests/full` + `crests/mini` barrels are unaffected.
- Every asset ships as BOTH svg and png ÔÇö there is no per-asset `delivery` field.
- `COMPONENT_SET` variants are flattened to one asset per variant; duplicate names get a
  `-2` suffix (`dedupeSlugs`, applied per tier). SVG ids are prefixed `<namespace>-<slug>-`
  so inlined assets never collide.
- **Renames** (`scripts/lib/renames.ts`): a global `RENAMES` map (`namespace` Ôćĺ original
  `slug` Ôćĺ `{ slug?, name? }`) gives poorly-named assets a better slug and/or display name ÔÇö
  e.g. `dadd-ai-l`/`dadd-ai-r` Ôćĺ `dadd-ai-left`/`dadd-ai-right`. Applied by `codegen` (read
  inputs by the Figma slug, emit every module/identifier/export/`AssetMeta` by the renamed
  one); the committed `assets/` mirror keeps the raw Figma slug, so `sync` never reconciles
  hand-edits. Offline (`pnpm codegen`); changes the public API, so commit a changeset with it.
- `tsc` runs with `isolatedDeclarations` ÔÇö every generated export needs an explicit type.
- Exports are fully namespaced with no back-compat, one family per export group `<g>`
  (`<g>` is `hoggies`, `crests/full`, or `crests/mini`): `./<g>`, `./<g>/svg[/*]`,
  `./<g>/png[/*]`, `./<g>/metadata`, plus `./crests` (combined compound barrel), `./logo`
  (the standalone `<Logo>` component ÔÇö no svg/png/metadata subpaths), `./colors`,
  and `./colors/css`. Root `.` is React-free and image-free: types, the cross-namespace
  `allAssets` manifest (`src/generated/manifest.ts`, the union of every group's
  `manifest.ts`), and the search helpers `findAssets`/`getAsset`/`getComponentName`
  (`src/search.ts`) ÔÇö safe to import when building an asset picker without pulling in any
  component or payload.

## Releases

No PRs ÔÇö everything is committed directly to `main` (verified API commits via the
Releaser GitHub App). Daily `sync.yml` (needs the `FIGMA_TOKEN` secret) commits new assets +
a changeset to `main`; that triggers `release.yml`, which (gated by the `Release`
environment + Slack approval, per the PostHog SDK handbook) runs `changeset version`, commits
the bump to `main`, and publishes to npm via trusted publishing (OIDC). Publishing
`@posthog/brand` requires the npm trusted-publishing config to be set up for that package name.

## Demo site

`site/` is a Vite + React SPA that showcases the whole package (overview, logo, colors, hoggies,
crests). The repo is a pnpm workspace (`pnpm-workspace.yaml`); `site/` is a separate private
package (`@posthog/brand-site`) that depends on `@posthog/brand` via `workspace:*` and imports it
through the real `exports` map (Ôćĺ `dist/`), so it doubles as an integration test of the published
surface. Because of that, the lib must be built first: `pnpm dev:site` / `pnpm build:site`
(root scripts) run `pnpm build` then the site. The asset grids render **live components** resolved
by slug (`getComponentName` + a namespace import of the barrel ÔÇö `site/src/assets-hoggies.ts`,
`assets-crests.ts`), and the heavy catalog routes are `React.lazy` code-split so the initial bundle
stays small. The site is themed from the package's own `colorsCss` tokens. It is NOT published to
npm; it deploys to Cloudflare Pages via the dashboard's Git integration (root dir = repo root,
build = `pnpm build:site`, output = `site/dist`, `NODE_VERSION=20`).
