# CLAUDE.md

`@posthog/hedgehogs` — an npm library that bundles PostHog's hedgehog illustrations as
React components, raw SVGs, and searchable metadata. Fully offline; no runtime CDN.

## Data flow

```
art-pipeline (public catalog)  --sync.ts-->  assets/  --codegen.ts-->  src/generated/  --tsdown-->  dist/
```

- **`scripts/sync.ts`** — the only networked script. Fetches `index.json` + `tags.json`
  from the art library, downloads a 768px PNG for every asset and the SVG for assets whose
  SVGO-optimized size is ≤ 512 KB, and writes `assets/`. Sends a browser `User-Agent`
  (r2.dev 403s otherwise). Run with `pnpm sync`.
- **`assets/`** — committed source of truth (`catalog.json`, `tags.json`, `sync-state.json`,
  `vectors/*.svg`, `png/*.png`). Sync PRs are reviewable here.
- **`scripts/codegen.ts`** — pure transform of `assets/` → `src/generated/`. Gitignored
  output; regenerated by `prebuild`/`pretest`. Run with `pnpm codegen`.
- **`src/`** — handwritten runtime + entry points; `src/generated/` is machine-written.

## Conventions

- Component name = `Hedgehog` + PascalCase(slug). SVG string = `hedgehog<Name>Svg`.
- Each asset's `delivery` is `"svg"` (bundled vector) or `"png"` (bundled raster). Goal:
  migrate everything to `"svg"`.
- `tsc` runs with `isolatedDeclarations` — every generated export needs an explicit type.

## Releases

No PRs — everything is committed directly to `main` (verified API commits via the
Releaser GitHub App). Daily `sync.yml` commits new art + a changeset to `main`; that
triggers `release.yml`, which (gated by the `Release` environment + Slack approval, per
the PostHog SDK handbook) runs `changeset version`, commits the bump to `main`, and
publishes to npm via trusted publishing (OIDC). `migration-report.yml` produces an
SVG-vs-PNG status report (job output + step summary) every Monday, only if there were
commits that week.
